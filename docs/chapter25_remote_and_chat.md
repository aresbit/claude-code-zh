## 第二十五章 远程操控与随处可用

上一章讲它爬出了终端，能在网页和手机上照看。这一章再进一步：人在外头，怎么**接着指挥**那只还在你电脑上跑着的怪兽。

### 一

第一招叫 **Remote Control（远程操控）**。

它和网页版有个根本区别：网页版跑在云端机器上，而远程操控——**怪兽还在你自己电脑上跑**，你的文件、你的 MCP 服务、你的项目配置，原封不动。网页和手机不过是一扇窥视它的窗户。

起一个远程会话，在项目目录里敲：

```bash
claude remote-control
```

它会停在那儿当「服务端」，给你一个会话网址；按一下空格还能显示二维码，手机扫一下直接连上。或者你不想另起，就在当前会话里：

```text
/remote-control
```

把眼下的对话历史一并带过去。（需要 v2.1.51 以上、claude.ai 订阅；Team/Enterprise 还得管理员先开开关。）

连上之后妙在哪？对话在所有设备间**同步**。你在终端敲一句，手机上也能看见；手机上回一句，终端里也接得上。笔记本睡着了、网断了，机器一回来它自动重连。

安全上也讲究：你的电脑只往外发 HTTPS 请求，**不开任何入站端口**，全程走 Anthropic API 的 TLS 通道。这点我特意去确认过——毕竟是把自己机器的会话往外接，不放心不行。

### 二

第二招叫 **Dispatch（派活）**。

人在外头，懒得连会话，就想甩一句「把那个 bug 修了」。用手机 App 给它发个任务，它能在你的桌面端起一个会话去办。适合「我懒得管细节，你看着办」的场景。

### 三

第三招叫 **Channels（频道）**。

这个有意思：把 Telegram、Discord、iMessage 这些聊天工具，或者你自己的 webhook，**接进**一个会话里。于是 CI 挂了、聊天群里有人喊一嗓子，怪兽就能闻声而动。它跑在你本机，靠装个 channel 插件接上。

### 四

第四招是 **Slack**。

在团队频道里 `@Claude` 一下，丢个 bug 报告过去，它能给你回一个 PR。这个跟前几招不同——它跑在 **Anthropic 云端**，要先装 Slack App 并开通网页版。适合一群人在群里协作的路子。

### 五

这几招容易混，我给你列个明白账，按「谁来触发、在哪儿跑」分：

| 招式 | 谁触发 | 在哪儿跑 |
|---|---|---|
| Dispatch | 手机 App 发任务 | 你的电脑（桌面端） |
| Remote Control | 从网页/手机驱动一个在跑的会话 | 你的电脑（CLI / VS Code） |
| Channels | 聊天 App 或你的服务器推事件 | 你的电脑（CLI） |
| Slack | 群里 @Claude | Anthropic 云端 |
| 定时任务 | 定个点 | CLI / 桌面 / 云端 |

（这些大多还是 research preview，开关、限额、行为都可能变。）

### 六

我想起从前在乡镇，乡长下村，靠的是一部老式对讲机，「滋啦滋啦」地喊。如今这只怪兽，你在天涯海角掏出手机，都能跟它对上话，还不带杂音。

但工具越是无处不在，越得想清楚一件事：**你到底要它干嘛。**对讲机那头吼一句话容易，吼错了，乡里的活儿也就办拧了。怪兽这头更是如此——它办事麻利，可办的是不是你真想要的，全在你那一句话。
