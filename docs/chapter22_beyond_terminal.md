## 第二十二章 走出终端：Web、桌面端与手机

我一直以为这只怪兽是终端里的住户，蹲在那一方黑底白字的小天地里，哪儿也不去。后来某天我打开浏览器，发现它已经从那口井里爬了出来——终端、IDE、桌面应用、网页、甚至手机，处处都有它的身影。

### 一

先说**网页版**。在浏览器里打开 `claude.ai/code`，你就能把任务丢给它，它在 Anthropic 自家的云端机器上干活。这事儿的妙处在于：你合上笔记本，会话还在跑；你回家路上掏出手机，还能看它干到哪一步了。

它干活的地方是一个**沙箱**——一个隔离的云端环境，预装了常用工具，默认只允许访问一批可信的包管理源和常见开发域名，别的网址一律 `403` 挡在门外。你要让它访问自己的内网服务，得先去环境设置里把域名加进白名单。

这功能眼下还挂着「research preview（研究预览）」的牌子，面向 Pro、Max、Team 用户开放。所谓预览，就是说它好用，但偶尔会抽风，你别太较真。

### 二

再说**桌面应用**。这是个独立的程序，macOS 和 Windows 都有。它的好处是可视化——改了哪些文件，diff 摆在眼前，红红绿绿，一目了然。你还能同时开好几个会话，左边一个右边一个，像同时使唤好几个伙计。

我这种老派的人，一开始是抵触图形界面的。终端多干净啊，一个黑框框，何必花里胡哨。但用过之后我承认，看 diff 这件事，图形界面确实比在终端里翻来翻去舒服。

### 三

最让我意外的是**手机**。

Claude 的手机 App（iOS 和 Android 都有），点进去有个 **Code** 标签，那就是你云端跑着的会话。我有回在地铁上，盯着手机看那只怪兽吭哧吭哧改一个我早上没写完的 bug，旁边大爷以为我在打游戏。

### 四

这几个地方不是各管各的，它们**底层是同一个引擎**。你的 `CLAUDE.md`、你的设置、你接的那些 MCP 服务，在哪个端都通用。

更妙的是会话可以**搬家**。你在网页或手机上起了个长任务，回到电脑前，一句：

```bash
claude --teleport
```

就能把那个会话「瞬移」回你的终端，接着干。反过来，本地的活儿想丢到云上去跑，也有 `--remote`。（`--teleport` 需要 claude.ai 订阅。）

### 五

我想起当年在机关，一件事得在好几个科室之间来回跑，盖章、签字、转单，跑断腿。现在这只怪兽倒好，一个会话从终端跑到云上，再跑到我兜里的手机上，连个章都不用盖。

工具进化到这一步，已经不是「在哪儿用」的问题了，而是「我人在哪儿，它就在哪儿」。这话听着有点吓人，但你慢慢会习惯的——就像当年我们习惯了手机一样。
