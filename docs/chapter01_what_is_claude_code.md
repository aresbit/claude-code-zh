## 第一章 我们谈论 Claude Code 时，究竟在谈论什么

二十七岁那年，我以为自己这辈子就这样了——守着一方终端，敲些枯燥的命令，敲到老。后来才晓得，日子总会冷不丁塞给你点意外的东西。比如那只叫 Claude Code 的怪兽，它没打招呼，就住进了我的终端。

那时我还年轻，把 AI 想得稀奇古怪。在我看来，它要么是浏览器里陪你闲扯的那种，要么是缩在编辑器角落、偷偷替你补两行代码的小精灵。见着 Claude Code，才知道不是这么回事。

### 一

Claude Code 是什么？

官方眼下的说法是：一个能读懂你代码库、会改文件、会跑命令、还能接上你各路开发工具的智能体；终端、IDE、桌面、浏览器，处处都使得。早些年它只蹲在终端那口井里，如今爬了出来，四处安家。不过我还是愿意把它当一只怪兽看——它不吃人，只吞代码和难题。你把难题喂进去，它在那儿琢磨一阵，再把答案吐出来。

那阵子我用 VS Code，也试过 Cursor，还让 GitHub Copilot 在我写代码时杵在边上指点。这些都不坏，像个热心的街坊，干活时凑过来念叨一句：「我看你该这么写。」

可 Claude Code 不是这路数。

它不问你要不要帮忙，上手就干。你撂下一句「把这个功能加上」，它当真去翻代码、列计划、改文件、跑命令，跑岔了还自己回头补救。这让我想起从前一块儿共事的一位老师傅，话不多，可你交代的事，他总给你办得妥妥帖帖。

### 二

那些年，我们这些写代码的有个通病：工具太多。

用 VS Code，得记它的快捷键；用 Vim，得背那一串莫名其妙的按键；至于 Emacs——罢了，用 Emacs 的人自有一套信仰，咱不去搅扰。

Claude Code 的好就好在，它不掺和这些。它蹲在终端里，像条看门狗。你使什么都行，VS Code 也好，Ghostty、Emacs 也罢，连 PowerShell 都成，只要你能开一个终端，它就在那儿候着。

最好的工具，是让你忘了它还在。Claude Code 大抵就是这样。

### 三

可天下没有白来的好处。

想用 Claude Code，得先给 Anthropic 交份月供。最便宜的一档叫 Pro，二十美元一个月——说贵不贵，说贱不贱，也就几杯咖啡的钱。只是话说回来，这条船一上，你就别想下来了。

往上还有 Max、Team、Enterprise，像茶楼里不同价位的包间，钱给得多，能使唤的次数也多。我起先用 Pro，没多久就嫌不够使，跟烟瘾上来总怪烟少是一个道理。

### 四

说到这儿，得提一桩顶要紧的事：上下文窗口（context window）。

官方文档开篇就告诫：Claude 的上下文窗口很容易塞满，一满，活儿就干得糙了。

这词儿听着唬人，掰开了说就一句话：这怪兽记性不济。你跟它聊得越久，它越容易丢三落四。起头你还记得自己为何而来，聊到后头，它就开始胡言乱语。

这让我想起在东北乡下当选调生那会儿，机关门房有个老头，记性差得出奇。今儿托他的事，明儿准忘个干净。Claude Code 也是这副脾气：刚说的话它记得，可你一股脑塞太多，它立马犯迷糊。所以使它有个诀窍——少绕弯子，直奔正题，该清场就敲一句 `/clear`，把最要紧的递给它就行，别指望它把你项目的陈年旧账全背下来。

有人给这起了个体面名号，叫「上下文工程」（context engineering）。说穿了，不过是学会怎么跟这只怪兽搭话：让它记住该记的，忘掉该忘的。

### 五

说到底，Claude Code 这东西，使顺了是宝贝，使拧了是麻烦。它不像那些动不动就替你出主意的 AI；它更像个闷头干活的伙计，你支使它做什么，它就做什么——哪怕你让它一头撞墙，它也认认真真撞给你看。

所以用它的头一条要诀是：先想明白自己要什么。别盼着它提醒你「这样不妥」，它只照你说的办，至于后果，那是你自个儿的事。

二十七岁那年，我以为握住了工具就握住了世界。后来才慢慢懂得，工具终归是工具，要紧的是握工具的人。Claude Code 是只厉害的怪兽，可牵着它走的，是你的手、你的心、你的脑子。
