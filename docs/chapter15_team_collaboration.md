## 第十五章 团队协作：一群人与一群智能怪兽

我年轻时在一家小公司做事，那会儿统共没几个人，挤一间办公室，有事吼一嗓子就齐了。后来公司大了，人多了，散在不同的城市，乃至不同的国家，协作这事就一下子复杂起来。

拿 Claude Code 做团队协作，也是这么个味儿。一个人用，和一群人用，是两码事。

### 一

一队人用 Claude Code，头一件要了断的，是配置怎么共。

各人揣各人的 CLAUDE.md，写出来的代码就一人一个样：有人用单引号，有人用双引号；有人爱加分号，有人嫌它碍眼；有人用 Tab 缩进，有人用空格。到末了，代码库就成了一锅大杂烩。

所以一队人该共一份 CLAUDE.md：搁项目根目录，提交进版本库，众人守同一套规矩。

### 二

再一件，是 Skills 怎么共。

一队人可以攒一套共用的技能，搁项目的 `.claude/skills/` 里。譬如代码审查的、生成测试的、写文档的、走发布流程的。如此一来，大家用同一把尺子，出来的活也就一个调门。企业用户还能在管理后台配组织级的技能，全公司的人都使得。

### 三

还有，是权限怎么管。

一队人里头，不该人人一样的权。新手兴许得多搁几道确认，老手可以多几分自主。

这类组织级的管束，靠的是「托管设置」（managed settings）这一层——管理员把规矩钉死在那儿，谁能跑哪些命令、谁够得着哪些连接、谁动得了哪些文件，下头的个人配置盖不过它。这就像公司的门禁：不同级别的人，进不同的区。

### 四

一队人用 Claude Code，还有桩绕不开的事：花销。

Claude Code 是按 token 算钱的。一个人用还好，十几个人一齐用，账面就上去了。

所以得把成本看住：

- 定好用量的上限
- 盯着使用的情形
- 把提示词打磨利索，省下不必要的 token
- 琢磨琢磨 Team 或 Enterprise 套餐，兴许更划算

这就像公司的办公用品：得有预算，不能由着人无限制地领。

### 五

总之，一队人用 Claude Code，比一个人用要费心得多。

要共配置、齐标准、管权限、控花销。可这些一旦理顺了，整队人的效率能上一大台阶。这就像一支乐队：人人手里一件乐器，可只有照同一份谱、踩同一个点，才奏得出和谐的曲子。
