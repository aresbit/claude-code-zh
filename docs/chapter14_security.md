## 第十四章 权限与安全：与智能怪兽相处的界限

我年轻时养过一条狗，叫大黄。大黄挺听话，可也闯祸。有回我没看住，它把邻居家的鸡咬了。打那以后我就明白：养狗得有规矩，不能由它乱窜，该拴的时候得拴。

用 Claude Code 也是一个理儿。它本事大，本事大就意味着风险大。你得晓得怎么给它划界限，免得它闯祸。

### 一

Claude Code 都有哪些权？

不另设防的话，它能：

- 读你项目里所有的文件
- 改你项目里所有的文件
- 跑 shell 命令
- 用你配好的 MCP 连接

这权头已经不小了。它读得了写得了你的代码，跑得了任意命令，真要由着它，连「删库跑路」都做得出来。

### 二

所以头一条铁律：看清楚了再点头。

碰上要紧的动作，Claude Code 多半会先问你一声：

```
我要删掉文件 src/old.js，行吗？
我要跑命令 rm -rf node_modules，行吗？
```

别顺手就敲个「y」。看明白它要干什么，确认没岔子了再应。养成这习惯，能躲过好些惨剧。

### 三

再一条，是权限分级。

Claude Code 有几档不同的权限模式，按一下 `Shift+Tab` 就能在它们之间倒：

- **严着来**：每跑一条命令都问一遍
- **寻常**：危险的动作问，寻常的动作自己做
- **放开手**：多半动作自己做，只有特别凶险的才问

新手我劝你先严着来，摸熟了再换寻常档。别轻易放开手，除非你真清楚自己在做什么。

### 四

还有 MCP 连接的安全。

前头说过，MCP 让 Claude Code 够得着外头的服务。可这也意味着：万一它被人做了手脚，就可能借这些连接干坏事。

所以配 MCP 要守一条「够用就好」的本分：

- 只给用得着的权
- 生产环境用只读的连接
- 要紧的动作搁道二次确认
- 时常翻翻 MCP 的使用日志

这就像你家的钥匙：不是谁都能给的，给了也得讲清楚能开哪几道门。

### 五

末了一条：机密别往代码里写。

API 密钥、数据库口令、私钥，这些东西别硬写进代码。Claude Code 看得见你的代码，你把这些埋进去，它就可能给抖搂出去。

正经的做法是：

- 用环境变量
- 用配置文件，但把它塞进 `.gitignore`
- 用专门的密钥托管服务

总之，安全这根弦，得从头绷到尾。这只怪兽本事大，可大本事得小心使。划好界限，留着戒心，才好安安稳稳地享它的便利。
