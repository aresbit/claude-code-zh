## 第二章 安装与初次见面

遇见 Claude Code 的那个下午，天晴得很，窗外的麻雀在树上叽叽喳喳。我坐在终端前，像个要去相亲的后生，又盼又怵。

### 一

安装这桩事，说简单也简单，说麻烦也麻烦。

官方眼下最推荐的，是一行原生脚本——不靠 Node.js，装完还会在背后自个儿更新：

```bash
# macOS / Linux / WSL
curl -fsSL https://claude.ai/install.sh | bash
```

```powershell
# Windows PowerShell
irm https://claude.ai/install.ps1 | iex
```

你要是认 Homebrew 或 WinGet，也有现成的包：

```bash
brew install --cask claude-code        # macOS
winget install Anthropic.ClaudeCode    # Windows
```

至于老派些、手头已经备着 Node.js（18 以上）的人，走 npm 这条道也成——只是包名是 `@anthropic-ai/claude-code`，当心别漏了当中那个 `-ai`。我当年就栽在这上头，对着一个 `404` 发了半天愣：

```bash
npm install -g @anthropic-ai/claude-code
```

npm 这东西，你是知道的，总爱出些莫名其妙的岔子。一会儿权限不够，一会儿网络不通，一会儿干脆就是它自个儿犯了倔。官方把原生脚本摆头一位，不是没缘由的。

我那回装得还算顺当，许是事先拜过电脑爷的缘故。装完先敲一句 `claude --version`，看它还喘不喘气；再敲一个 `claude`，那只怪兽就醒了。万一装是装上了却叫不应，`claude doctor` 会替你搭个脉。

### 二

头一回启动，它要你登录。如今 Anthropic 用的是网页登录：弹出一个浏览器窗口，你点几下授权就完了。这让我想起从前那些张口就问你要 API key 的工具——你得上网站翻出一长串乱码似的字符，抄过来还提心吊胆，生怕错一个字母。

如今总算省心了些。

登录罢，它问你想在哪个目录里干活。现成的项目也行，新起一个也行。我挑了个平日写代码的目录，里头堆着各色半截子的实验项目，活像个杂货铺。

### 三

头一回正经说话，我竟有些紧张。

不知道开口说什么好，跟头回相亲不知从何聊起一个样。憋了半天，我憋出俩字：「你好。」

它回了一大段，无非「我是 Claude，很高兴帮你」之类的场面话。我有点失望——本指望它有点脾性，比方甩一句「哟，新来的？」。

后来才晓得，它的脾性不在开场白里，而在那份系统提示里。那东西改一改，它的性子也就跟着变了。

### 四

给项目开张时，有个很顶用的命令：`/init`。

它会让 Claude Code 把你的项目结构摸一遍，再生成一个 CLAUDE.md。这文件要紧得很，是你跟这只怪兽立的字据：你把项目的规矩、约定、忌讳写进去，往后它说话办事，就照着这些来。

我拿一个项目试了试，它生成的 CLAUDE.md 还挺像样，项目结构、技术栈、常用命令，一条条列得明白。当然，后来我又按自己的性子改了不少。

这就跟养狗一个理儿：先得立规矩——哪儿能撒尿，哪儿不许咬。

### 五

安装、配置都妥当了，我才算正经用上了 Claude Code。

那会儿我还不晓得，这只怪兽往后会怎样翻搅我的活法。我只当它是个更好使的搜索引擎，或者一个会写代码的话痨。可没多久就发觉，它比我想的要复杂得多，也厉害得多。

它像一把快刀，使得好，能切菜能雕花；使不好，先割了自己的手。怎么把这把刀使利落，正是我接下来要讲的故事。
