# 玩转通用智能体之Claude Code

## 第二十四章 自动记忆：让它记住该记住的

前面第一章我抱怨过，说这怪兽记性不好，聊多了就迷糊。这话现在得补一句：它学会**自己记笔记**了。

### 一

得先分清两样东西，它们都在每次会话开头就加载进来，但出处不同：

- **CLAUDE.md**：你写的。规矩、约定、构建命令、"永远要做 X"——你想让它记住的，你自己敲进去。
- **自动记忆（Auto memory）**：它自己写的。它一边干活一边把学到的东西记下来：这个项目用什么命令构建、那个 bug 当时是怎么定位的、你偏好用 pnpm 不用 npm……

一句话：**CLAUDE.md 是你立的规矩，自动记忆是它自己悟出来的心得。**

（自动记忆需要 Claude Code v2.1.59 以上，`claude --version` 查一下。默认是开着的。）

### 二

它不是每次都记。它会掂量：这条信息，将来的对话里用得上吗？用得上才记。

记在哪儿呢？每个项目有自己的一个目录：

```text
~/.claude/projects/<项目>/memory/
├── MEMORY.md          # 一份简明索引，每次会话都加载
├── debugging.md       # 调试心得的详细笔记
├── api-conventions.md # API 设计上的决定
└── ...                # 它自己开的别的主题文件
```

那个 `<项目>` 是按 git 仓库算的，所以同一个仓库的所有 worktree、所有子目录，**共用一份记忆**。

### 三

机制上有个细节值得知道：每次会话开头，只加载 `MEMORY.md` 的**前 200 行、或前 25KB**，谁先到算谁。所以它把 `MEMORY.md` 当索引使，详细的东西挪到 `debugging.md` 这类主题文件里，用到的时候才现读。

这设计挺聪明——记得多，但不一股脑全塞进脑子里占地方。这和我前面讲的 context window 那点道理是一脉相承的：好的记性不是什么都记在前台，而是知道什么放前台、什么搁后头、用时再翻。

界面上你要是看到"Writing memory"或"Recalled memory"，那就是它在动这个目录了。

### 四

这些都是**纯 markdown 文件**，你随时能看、能改、能删。会话里敲：

```text
/memory
```

就能列出当前加载的所有 CLAUDE.md、CLAUDE.local.md、规则文件，还能开关自动记忆，以及打开那个记忆文件夹。

你直接跟它说"以后都用 pnpm，别用 npm"或者"记住跑 API 测试得先起个本地 Redis"，它就给你存进自动记忆里了。想存进 CLAUDE.md 的话，明说"把这条加到 CLAUDE.md 里"就行。

### 五

要关掉自动记忆，`/memory` 里有开关，或者在项目设置里：

```json
{
  "autoMemoryEnabled": false
}
```

也可以用环境变量 `CLAUDE_CODE_DISABLE_AUTO_MEMORY=1`。

不过我劝你别急着关。我用了一阵子发现，它记的那些东西，恰恰是我懒得写进 CLAUDE.md、又总在重复解释的边角料。它替我记了，我省了一遍遍唠叨的功夫。

### 六

我那个记性差的门卫老头，要是当年也有个这样的小本本，兴许就不会把我托办的事忘得一干二净了。

机器的好处就在这儿：它不嫌记笔记麻烦，也不会觉得"这点小事还用记"。该记的记下，该忘的——你删了它就忘了。比人靠谱。
