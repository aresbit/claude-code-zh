# 玩转通用智能体之Claude Code

## 第十六章 CLI 工具：终端里的七十二变

我年轻时很服那些能在终端里飞檐走壁的人。他们敲几下命令，屏上的字飞快地滚，一桩桩繁难的活转眼就办妥了。我看着觉着真带劲，活像个武林高手。

后来才懂，那些看着神乎其神的把式，无非是工具使对了。每件工具有每件的用场，高手晓得什么时候抄什么家伙。

Claude Code 自个儿就是件 CLI 工具，可它不光自己好使，还能帮你把别的 CLI 工具使利索。

### 一

什么是 CLI 工具？

CLI 是 Command Line Interface，命令行界面；跟它对着的是 GUI，图形界面。

CLI 工具就是那些在终端里跑的程序——`ls`、`cat`、`grep`、`git`、`npm`，都是。Claude Code 自个儿也是一件，你在终端里敲命令跟它打交道。

### 二

Claude Code 怎么帮你使 CLI 工具？

头一样，它能听懂你的大白话，再翻成对路的命令。

你说「瞧瞧当前目录下都有些什么文件」，它就晓得该跑 `ls -la`。你说「把所有带 TODO 的 JavaScript 文件揪出来」，它就晓得该跑 `grep -r "TODO" --include="*.js" .`。

这对记不住命令的人，实在友好——你不必背那一串古怪的参数，只消讲清楚想干什么。

### 三

第二样，它能替你掰扯命令的输出。

有时一条命令跑下来，吐出一大片，看得人眼晕。Claude Code 能替你把这堆东西理一理，把要紧的拎出来，换个清爽的样子摆给你。

譬如你跑 `git log`，出来几十条提交记录。你问它一句「最近一礼拜有哪些要紧的改动？」它就替你筛、替你拢，把重点点给你。

### 四

再一样，它能替你写脚本。

要是有一串命令你常要跑，不如写成脚本。Claude Code 能替你写，连每一行是干什么的都给你讲明白。

你说「帮我写个脚本，自动备份数据库、打包、传到云存储上」，它就能给你来一段齐整的 bash 脚本，错误处理、日志记录，一应俱全。

### 五

可 CLI 工具也是要学的。

Claude Code 帮得上忙，但你要是对 CLI 一窍不通，仍会觉着吃力。我的主张是：

- 把最常用的几个先学会：`ls`、`cd`、`cat`、`grep`、`git`
- 把几个基本的门道弄懂：管道、重定向、环境变量
- 别什么都指着 Claude Code，自己也得长本事

这就像学开车。自动挡是省事，可你到底得懂些车况、懂些路规，不能把命全交给车自个儿开。

总之，CLI 工具是写代码的人的看家本领，Claude Code 能让它们更好使，可你自己也得下点功夫。把这些家伙使熟了，你也就能在终端里飞檐走壁了。
