# 玩转通用智能体之Claude Code

## 第八章 Hooks：自动化的把式

我年轻时在厂里做过工，那阵子最眼馋的，是那些自动化的生产线。按一个钮，机器自个儿就转起来，该切的切，该焊的焊，末了成品就出来了。人只消在旁边照看，偶尔料理一下岔子。

Claude Code 的 Hooks，就有几分这个意思。

### 一

Hooks 是什么？

说白了，就是在某些特定的当口，自动替你跑一段命令。譬如：

- 每回会话一开张，自动查一遍环境
- 每回动文件之前，先备个份
- 每回跑完命令，记一笔日志

这就像在你的流程里埋下几个机关，条件一到，相应的动作就自个儿触发。

### 二

要紧的是得分清：Hooks 不是写在 CLAUDE.md 里的「嘱咐」，而是钉死在生命周期事件上的 shell 命令。

CLAUDE.md 那套，是讲给怪兽听的道理——它听不听、听几分，没准儿。Hooks 不一样，它是实打实的命令，挂在 `settings.json` 里，到了点（会话开始 `SessionStart`、动工具前 `PreToolUse`、动完之后 `PostToolUse` 等等）就铁板钉钉地执行，由不得怪兽改主意。

也正因如此，凡是「非这么干不可」的事——提交前必跑测试、改完文件必走一遍格式化——交给 Hooks，比写进 CLAUDE.md 里求它照办，靠谱得多。

### 三

Hooks 的好处在哪？

头一桩是省心。那些反复要做的杂事——查环境、备份、记日志——不必每回伸手，撂给 Hooks 自动料理，你专心去啃真正的难题就是。

第二桩是齐整。一队人都挂同一套 Hooks，流程就拉齐了，不会再冒出「我忘了查环境」「我又忘了备份」这类话。

第三桩是稳。要紧的动作前头搁道关卡——提交前过一遍测试，部署前验一验配置——能挡掉好些低级的纰漏。

### 四

可 Hooks 也有它的累赘。

最显眼的是慢。每个动作前头都先跑一摞检查，积少成多，整体就拖沓了。这跟过安检一个样：为的是稳妥，到底添了麻烦。

再一个是绕。Hooks 一多，你常闹不清是哪一个在作怪，出了岔子不好查。好比家里电器太多，遥控器堆一摞，分不清哪个管哪个。

所以我的经验是：Hooks 要有，但别贪多。只留那些真要紧、能挡大祸的；锦上添花的，能省则省。

### 五

总之，Hooks 是 Claude Code 的一手高级把式，使得好，能让你的流程更自动、更齐整。可也别使过了头——保持简单，保持可控，才是正道。
