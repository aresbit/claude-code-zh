# 玩转通用智能体之Claude Code

## 第十章 MCP：连接世界的桥梁

我年轻时有个念想：要是有个万能遥控器，把家里的电器一手攥着该多好——电视、空调、音响、电灯，一个钮全管了。后来智能家居真成了，我又嫌不够，还想拿这遥控器管更多的东西：我的车，我的办公室，乃至街口的红绿灯。

MCP（Model Context Protocol，模型上下文协议）大致就是 Claude Code 的那只「万能遥控器」。

### 一

MCP 是什么？

它是 Claude Code 跟外头世界打交道的一套协议。借着它，你能让 Claude Code 接上各样的外部服务：数据库、API、文件系统、消息队列，等等。

没有 MCP 那阵子，Claude Code 只摆弄得了你本机的文件和命令。有了 MCP，它便能跟整个世界搭上话——替你查数据库、调第三方 API、读写远端的文件，乃至指挥别的软件。

这就像从一间闭塞的屋子里走出来，迈进了宽敞的天地。

### 二

怎么把 MCP 接上？

最省事的是一句命令。譬如接一个服务：

```bash
claude mcp add <名字> -- <启动这个服务的命令>
```

接好的服务，也可以写进项目根目录的 `.mcp.json` 里，随仓库一道带走，一队人共用一套。配妥之后，你就能直接跟它说话了：「把数据库里用户表的结构查一查」，或者「调个接口，取一下今儿的天气」。

### 三

MCP 把 Claude Code 的能耐撑大了一大圈。

从前你要查数据库，得自个儿开客户端、敲 SQL、瞅结果。如今一句话的事，它替你办了，还把结果归置得清清爽爽。

从前你要调接口，得写代码、跑脚本、料理返回的数据。如今它能直接替你试接口、剖数据，乃至替你把调用的代码也写了。

这就像身边多了个全能的帮手：不光会写代码，连各样工具都替你使唤，省下你好些工夫。

### 四

可 MCP 也带着安全的隐患。

你给了它进数据库的钥匙，它就读得了数据；配得不当，它兴许还能改数据、删数据。你给了它调接口的权，它就调得动那接口；接口要是按次扣费的，使猛了就是真金白银。

所以配 MCP 得留神：

- 只给用得着的权，能只读就别给写
- 要紧的动作搁道确认，别一路全自动
- 时常回头查查 MCP 的使用，看有没有蹊跷

这就像把家门钥匙交给了保姆：得信她，可也得留个心眼。

### 五

总之，MCP 是 Claude Code 的一手厉害功夫，它让你这个 AI 帮手不再困在本机的代码里，而能跟整个世界交手。使得好，能替你办成许多事；使不好，也招得来风险。配得审慎，使得在理，才掂得出它的分量。
