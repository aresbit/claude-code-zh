# 玩转通用智能体之Claude Code

## 第二章 安装与初次见面

我遇见Claude Code的那个下午，天气晴朗，窗外的麻雀在树上叽叽喳喳。我坐在终端前，像个即将相亲的年轻人，既期待又忐忑。

### 一

安装这件事，说简单也简单，说复杂也复杂。

官方现在最推荐的，是一行原生安装脚本——它不依赖 Node.js，装完还会在后台自己更新：

```bash
# macOS / Linux / WSL
curl -fsSL https://claude.ai/install.sh | bash
```

```powershell
# Windows PowerShell
irm https://claude.ai/install.ps1 | iex
```

如果你是 Homebrew 或 WinGet 的拥趸，也有现成的包：

```bash
brew install --cask claude-code        # macOS
winget install Anthropic.ClaudeCode    # Windows
```

当然，老派一点、手里已经有 Node.js（18 以上）的人，还是可以走 npm 这条路——注意包名是 `@anthropic-ai/claude-code`，别漏了中间那个 `-ai`，我当年就栽在这上头，对着 `404` 发了半天呆：

```bash
npm install -g @anthropic-ai/claude-code
```

npm这东西，你知道的，总是会有各种奇怪的问题。有时候是权限不够，有时候是网络不通，有时候干脆就是它自己抽风了。所以官方把原生脚本摆在第一位，不是没有道理。

我安装的时候倒还算顺利，可能是因为我提前拜了拜电脑之神。装完之后，先用一句 `claude --version` 确认它还活着，再在终端里输入 `claude`，那只智能怪兽就醒了。要是它装好了却叫不应，`claude doctor` 会替你把脉。

### 二

第一次启动，它会让你登录。 Anthropic现在用的是网页登录，弹出一个浏览器窗口，让你输入账号密码。这事儿让我想起以前那些要你输入API key的工具，那时候你得去网站上复制一长串乱七八糟的字符，生怕弄错一个字母。

现在好了，至少你还有人性化了一点 kimi code。

登录完之后，它会问你要在哪个目录工作。你可以选一个现有的项目，也可以新建一个。我选了一个我平时写代码的目录，那里面乱七八糟地堆着各种实验性的项目，像个杂货铺。

### 三

第一次正式对话，我有点紧张。

我不知道该说什么，就像第一次约会不知道聊什么话题一样。最后我说了句："你好。"

它回复了一大段，什么"我是Claude，很高兴帮助你"之类的客套话。这让我有点失望，我以为它会更有性格一点，比如来一句："哟，新来的？"

但后来我发现，它的性格不在开场白里，而在官方的系统提示里， 改完后soul也是可以改变的。

### 四

初始化项目的时候，有个很有用的命令：`/init`。

这个命令会让Claude Code分析你的项目结构，然后生成一个CLAUDE.md文件。这个文件很重要，它是你跟这只智能怪兽之间的契约。你在里面写上项目的规则、约定、注意事项，它就会在之后的对话中遵守这些规矩。

我让我的一个项目执行了这个命令，它生成的CLAUDE.md还挺像模像样的，列出了项目结构、技术栈、主要命令等等。当然，后来我根据自己的需要又修改了不少。

这就像是你养了一只狗，要先给它定规矩：哪里可以尿，哪里不能咬。

### 五

安装和配置都搞定之后，我正式开始使用Claude Code。

那时候我还不知道，这只智能怪兽将会如何改变我的工作方式。我以为它只是个更好用的搜索引擎，或者是个能写代码的聊天机器人。但很快我就发现，它比我想象的要复杂得多，也强大得多。

它就像是一把锋利的刀，用得好可以切菜雕花的，用不好可能会割伤自己。而学会使用它，正是接下来我要讲的故事。
