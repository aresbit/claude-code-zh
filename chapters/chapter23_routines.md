# 玩转通用智能体之Claude Code

## 第二十三章 定时任务：让怪兽自己上班

人这一辈子，总有些活儿是重复的、无聊的、又不得不干的。每天早上看一遍昨晚的 PR，每周扫一遍依赖有没有过期，每次部署完跑一遍冒烟测试——这些事儿，交给一只不知疲倦的怪兽再合适不过。

### 一

这功能叫 **Routines（例程）**。一个 routine，说白了就是把一套配置存下来：一段提示词、一个或几个代码仓库、一组连接器（connector），打包一次，往后自动跑。

它跑在 Anthropic 的云端机器上，所以你笔记本合上了、关机了、出去钓鱼了，它照样按点上班。这点和定时任务的本意是一致的——你不在，活儿也得有人干。

（同样，眼下是 research preview，面向开通了网页版的 Pro、Max、Team、Enterprise 用户。）

### 二

routine 怎么"触发"，有三种法子，还能混着用：

- **定时（Scheduled）**：按钟点来，每小时、每晚、每周，或者就指定将来某个时刻跑一次。
- **API**：给它一个专属的 HTTP 端点，你拿 bearer token 往那儿 POST 一下，它就醒了。监控系统报警、部署流水线跑完，都能这么戳它一下。
- **GitHub 事件**：仓库里有了动静——开了 PR、发了 release——它自动响应。

一个 routine 可以同时挂这三种触发器。比如一个审 PR 的例程，可以每晚跑一遍，可以被部署脚本戳醒，还可以见着新 PR 就上。

### 三

创建有三个地方：网页 `claude.ai/code/routines`、桌面应用、或者干脆在终端里。终端里最省事，一句话的事：

```text
/schedule daily PR review at 9am
```

或者来个一次性的：

```text
/schedule in 2 weeks, open a cleanup PR that removes the feature flag
```

它会把你说的自然语言时间换算成绝对时刻，跟你确认一遍再存。`/schedule list` 看全部，`/schedule update` 改，`/schedule run` 立刻跑一遍。

不过要注意：终端里的 `/schedule` 只能建**定时**触发的；想加 API 或 GitHub 触发器，得去网页上编辑。

### 四

有件事我得唠叨一句，免得你像我一样吃亏。

routine 跑起来是**全自动的云端会话，中途没有权限确认、没有人盯着**。它能跑 shell、能调你给它的连接器、能用仓库里的 skill。所以它能碰到什么，全看你给它配了什么：选了哪些仓库、开没开"允许推送任意分支"、环境的网络权限、挂了哪些连接器。

默认情况下它只能往 `claude/` 开头的分支推，碰不了你的 `main`——这是个保命的设计。我第一次没仔细看，差点把一堆连接器的写权限全留给了一个半夜自动跑的例程。后来学乖了：**用得着什么给什么，别图省事全勾上。**

还有个坑：run 列表里显示绿色，只代表会话"正常启动并退出了"，**不代表你交代的活儿干成了**。点进去看 transcript 才作数。网络被挡、工具缺失、任务本身失败，都藏在里头，不在那个绿点上。

### 五

跟 Routines 容易混的，还有两个轻量货：

- **`/loop`**：在一个开着的终端会话里，把某个提示词每隔几分钟重复跑一遍。适合"盯着点儿"的活，比如轮询部署状态。
- **桌面定时任务**：跑在你**本机**上，能直接碰你本地的文件和工具。建 routine 时选 **Local** 而不是 **Remote**，就是它。

云端的归云端，本地的归本地，按你的活儿在哪儿挑。

### 六

我年轻时在机关，最佩服那种"事情交出去就不用管了"的同事。你给他一件事，他记在小本本上，到点儿自己办，办完给你个交代。Routines 就是这么个小本本——只不过它记性比人好，也不会嫌活儿烦。

唯一要操心的，是你得把话说清楚。它半夜自己跑的时候，可没法回头问你"这个到底啥意思"。
