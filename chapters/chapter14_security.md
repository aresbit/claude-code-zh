# 玩转通用智能体之Claude Code

## 第十四章 权限与安全：与智能怪兽相处的界限

我年轻时养过一条狗，叫大黄。大黄很听话，但有时候也会闯祸。有一次我没看住它，它把邻居家的鸡给咬了。从那以后我就知道，养狗要有规矩，不能让它乱跑，该栓的时候得栓。

用Claude Code也是一样。它很强大，但强大意味着风险。你得知道怎么给它设界限，防止它闯祸。

### 一

Claude Code有哪些权限？

默认情况下，Claude Code可以：
- 读取你项目里的所有文件
- 修改你项目里的所有文件
- 运行shell命令
- 访问配置好的MCP连接

这些权限已经很大了。它可以读写你的代码，可以运行任意命令，甚至可以删库跑路——如果你让它这么做的话。

### 二

所以安全第一原则：永远看清楚再确认。

Claude Code在执行敏感操作之前，通常会问你确认。比如：

```
我要删除文件src/old.js，确认吗？
我要运行命令rm -rf node_modules，确认吗？
```

不要习惯性地打"y"。看清楚它要干什么，确认没问题了再同意。养成这个习惯，能避免很多悲剧。

### 三

其次是权限分级。

Claude Code有不同的权限模式：

- **严格模式**：每次执行命令都要确认
- **普通模式**：危险操作要确认，普通操作自动执行
- **宽松模式**：大部分操作自动执行，只有特别危险的要确认

新手建议用严格模式，熟悉了之后可以切换到普通模式。不要轻易用宽松模式，除非你真的知道自己在做什么。

### 四

还有MCP连接的安全。

前面说过，MCP让Claude Code能访问外部服务。但这也意味着，如果Claude Code被恶意利用，它可能通过这些连接做坏事。

所以配置MCP的时候要遵循最小权限原则：
- 只给必要的权限
- 生产环境用只读连接
- 敏感操作需要二次确认
- 定期检查MCP的使用日志

这就像是你家的钥匙，不是谁都能给的，给了也要说清楚能开哪些门。

### 五

最后，敏感信息不要写在代码里。

API密钥、数据库密码、私钥，这些东西不要硬编码在代码里。Claude Code能看到你的代码，如果你把这些信息写进去，它就可能泄露。

正确的做法是：
- 用环境变量
- 用配置文件，但把配置文件加入.gitignore
- 用专门的密钥管理服务

总之，安全意识要贯穿使用Claude Code的始终。这只智能怪兽很强大，但强大的力量需要谨慎使用。设好界限，保持警惕，才能安全地享受它带来的便利。
