# 玩转通用智能体之Claude Code

## 第九章 Subagents：分身有术

小时候看《西游记》，最眼馋孙悟空那身分身的本事。揪一把毫毛，吹口气，就变出几百个小猴子，一齐上阵厮杀。我那时想，我要也有这本事该多好：一个我去上学，一个我去打球，还留一个我在家睡觉。

后来长大了，晓得分身是哄人的。可用 Claude Code 时，我撞见了一桩相仿的事——子代理（Subagents）。

### 一

子代理是什么？

说白了，就是让 Claude Code 派出几个分身，各领一摊活，末了把结果汇拢上来。这就像个工头，把大活拆成小活，分给不同的伙计，再把各人的成果并到一处。

为什么要这么折腾？

还是上下文窗口闹的。前头说过，Claude Code 记性有限，活儿一复杂它就兜不住。可拆成几摊小活，各交一个分身去办，就绕开了这道坎。

### 二

怎么使分身？

`/agents` 这个命令，是料理子代理的总入口。你也可以在话里直接吩咐它：

```
分析一下这个项目的架构，分几个子代理并行去啃不同的模块。
```

它便自个儿派出几个分身：一个管前端，一个管后端，一个管数据库……各自把分到的模块摸一遍，末了拼成一张完整的架构图。

你还能给某一路分身立下专门的性子和章程，写成文件搁在 `.claude/agents/` 里——往后这类活，就有专人专办了。

### 三

分身还有一个用处：把风险圈起来。

有时你想让它做点担风险的事，譬如一口气改一大摞文件。搁主会话里做，万一砸了，整个会话的上下文都得跟着遭殃。

可要是派个分身，让它在隔开的地界里折腾，就算砸了，也只脏了那一个分身，主会话还是干净的。

这就像做化学实验：有些凶险的反应得搁通风橱里做，不能把一整间实验室都熏了。

### 四

但分身也不是万灵的。

头一桩是协调的工夫。几个分身一齐做，末了得把结果并到一处。这并的过程有时挺磨人，尤其当各人的结果打架的时候。

第二桩是耗费。每个分身都占一份算力，分身太多，反倒慢，甚至撞上用量的上限。

所以使分身要有个度。小活犯不上拆，大活才拆。一般说来，等你瞧见主会话开始犯迷糊了，或者活儿牵连的文件太多了，就该想到请分身了。

### 五

总之，子代理是 Claude Code 对付复杂活计的一手要紧本事。它让你也能像孙悟空一样分身有术，把大难题拆成小难题，各个击破。记一条：拆得在理，使得有度，才逼得出它最大的劲道。
