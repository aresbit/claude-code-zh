# 玩转通用智能体之Claude Code

## 第二十一章 延伸阅读：站在巨人的肩上

写到这儿，我想起年轻时读书的光景。那会儿没有互联网，寻一本书不易。每寻着一本好的，便如获至宝，翻来覆去地读，做笔记，把里头的养分一点点吸进去。

如今信息多了，找资料容易了，可挑拣反倒难了。东西太杂，分不清哪些是好的，哪些是白耗工夫的。

这一章，我荐几样靠谱的去处，帮你接着往深里走。

### 一

**官方的**

- 官方文档（`code.claude.com`）：最权威的一处，基础和最佳实践都在这儿。
- Anthropic 的博客：官方的技术文章，新功能和设计的心思，看这里。
- 官方的示例库：照着例子学，常比啃文档来得直观。

### 二

**社区的**

- GitHub 上的 Awesome Claude Code：社区拢起来的一份好资源清单。
- Reddit 的 r/ClaudeCode：用户的讨论区，各样的用法和疑难都能瞧见。
- Discord 社区：实时聊，有问题能很快有人搭话。

### 三

**几句实在的建议**

光看书看文档不够，更要紧的是动手：

- 从小项目起步，把基本的操作摸熟
- 一点点啃复杂的活，往自己的极限上够一够
- 把心得记下来，攒成自己的一套
- 跟人多聊，学学别人的路数

### 四

**得跟着更新**

Claude Code 走得快，新功能一茬接一茬。要想不落后，就得：

- 盯着官方的发布说明
- 隔三差五翻翻文档的更新
- 新功能拿来试试，摸清它能做什么

这一行变得快，不学，转眼就被甩下了。

### 五

末了我想说的是：资料终归是辅助，真本事是用出来的。

别一头扎进资料里，光看不练。Claude Code 是件工具，使得多了自然就熟。碰上难处再回头查，这样效率最高。

就像学游泳：教程看再多，不下水也是白搭。一头扎进去，扑腾起来，呛上几口，慢慢也就会了。
