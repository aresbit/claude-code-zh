# 玩转通用智能体之Claude Code

## 第七章 会话管理：记住与遗忘的艺术

我年轻时认得一个人，记性差得离奇。昨儿的事他能忘得一干二净，二十年前谁欠他五毛钱却记得牢牢的。我常想，人脑真是个古怪东西：该忘的记着，该记的忘了。

Claude Code 也是这副德行。

### 一

用久了你就摸出来：跟它聊得越久，它越糊涂。这不是它存心跟你别扭，是那个叫上下文窗口的东西在作祟。

说白了，就是它记性有限。你俩聊了一整天，你前前后后提了上百个文件，可它真正攥得住的，兴许只剩最近那二三十个。再往前的？早被挤到爪哇国去了。

这让我想起小时候玩的传话游戏。头一个人说「今天天气很好」，传到第十个人嘴里，就成了「今天气死很耗」。话就是这样，传一道丢一点，到末了面目全非。

### 二

所以用 Claude Code，你得学会理会话。

最要紧的一手，是晓得什么时候该清场重来。一旦发觉它开始胡咧咧，或者把你前头交代过的要紧事忘了，就该寻思重启了。

清场的法子，敲一句 `/clear`，当前会话就清干净了，从头开张。要是连人带摊子都不想要了，敲 `/exit` 走人，下回再起一个新的。

这就跟收拾屋子一个理儿：东西堆杂了就乱，不如一股脑清空，重新归置。是费点事，可清爽。

### 三

可有时候你不想全推倒，因为前头聊的还有用。

这当口可以使 `/compact`。这命令会让它把会话的来龙去脉压一压，撇开那些不打紧的，腾出地方来。

像是清理硬盘：删掉临时文件、倒空回收站，地方就回来了。虽不及全新会话那般清爽，好歹能接着往下使。

### 四

还有一种情形：活儿太大，一个会话装不下。

譬如要把一个超大项目的架构摸个透，兴许牵连几百个文件。这种当口，与其硬塞给一个会话，不如拆成几摊子小活，每摊交给一个单独的子代理去料理。

这就像一家大公司，老总记不住每个伙计的细枝末节。他把事派给部门头儿，部门头儿再派给组长，层层分下去，末了把结果汇拢上来。

Claude Code 的子代理（Subagents）就是干这个的。下一章细讲。

### 五

总之，理会话是用好 Claude Code 的一门要紧功夫。

记住这几条：

- 瞧它开始犯糊涂了，就清场重来
- 还想留点上下文，就使 `/compact`
- 活儿太大，就拆给子代理

这跟养牲口一样：你得摸熟它的性子，晓得它什么时候乏了、什么时候饿了、什么时候该牵出去遛遛。摸透了它，才使得顺手。
