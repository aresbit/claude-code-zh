# 玩转通用智能体之Claude Code

## 第一章 当我们在谈论 Claude Code的时候， 我们在谈论什么？

我二十七岁那年，以为会一辈子在终端里敲着枯燥的命令。后来才知道，生活总会给你一些意想不到的礼物，就像那只叫做Claude Code的智能怪兽，它悄无声息地出现在我的终端里。

那时候我还年轻，对于AI这件事有着种种奇怪的想象。我以为AI就是那些在浏览器里与你聊天的东西，或者躲在编辑器角落里偷偷给你补全代码的小精灵。直到我遇见了Claude Code，才发现事情并非如此简单。

### 一

Claude Code是什么？

官方现在的说法是："一个能读懂你代码库、改文件、跑命令、还能接上你各种开发工具的智能体编程工具——终端、IDE、桌面端、浏览器里都能用。"早些年它只蹲在终端里，如今已经从那口井里爬了出来，四处安家。但我还是更愿意把它想象成一只怪兽，它不吃人，只吞噬代码和问题。你把问题喂给它，它就在那儿思考，然后吐出答案来。

那时候我还在用VS Code，也试过Cursor，甚至让GitHub Copilot在我写代码的时候在旁边指手画脚。这些都挺好，就像有个热心肠的邻居，总在你干活的时候凑过来说："我觉得你应该这样写。"

但Claude Code不一样。

它不问你要不要帮忙，它直接动手。你告诉它："把这个功能加上去。"它就真的去翻你的代码，做计划，改文件，跑命令，出错了还自己修。这让我想起我参加选调时候认识的一个女研究生，话不多，但交代他的事总能给你办得妥妥帖帖。

### 二

那年月，我们这些写代码的人都有一个共同的烦恼：工具太多了。

你要用VS Code，就得学它的快捷键；用Vim，就得记住那些莫名其妙的按键组合；用Emacs——算了，用Emacs的人有自己的宗教，我们不说他们。

Claude Code的好处在于，它不管这些。它就蹲在终端里，像只看门狗。你用什么都行，VS Code、Ghostty、Emacs，甚至是powershell，只要你能打开终端，它就在那儿等着。

"最好的工具，就是让你忘记它的存在。"Claude Code差不多就是这样。

### 三

但凡事都有代价。

要用Claude Code，你得先给Anthropic交保护费。最便宜的套餐是20美元一个月，叫做Pro 会员。这价钱说贵不贵，说便宜也不便宜，差不多就是几杯咖啡的钱。但你要知道，一旦上了这条贼船，你就再也回不去了。

还有更贵的套餐：Max、Team、Enterprise。这些就像是不同档次的包间，越贵能用的次数越多。我一开始用的是Pro，后来发现根本不够用，就像抽烟的人总觉得烟不够抽一样。

但凡事也有变数，

你也可以使用kimi登陆。

甚至你也可以定制你的 CC工具， 打个广告 ，想使用定制化可破解 自由使用的CC可以联系我。

### 四

说到这儿，我还得提一个最重要的事情：context window。

官方文档开篇就说："Claude的context window很容易满，满了之后性能就会下降。"

官方没有告诉你的是如果你使用其他大模型供应商，你的缓存会在5 min失效。

context window这话听起来很技术，但说白了就是：这智能怪兽的记性不太好。你跟它聊得越多，它就越容易忘事。一开始你还记得你是为什么来找它的，聊到后面，它就开始胡言乱语了。

这让我想起我当年在东北农村选调当公务员的时候，机关里有个门卫老头，记性差得离谱。你今天问他的事，明天他就忘了。Claude Code也差不多。它记得住你刚说的话，但你要是一口气给它塞太多东西，它就开始迷糊了。所以用它的诀窍就是：少说废话，直奔主题，多多/clear。 给它最必要的信息，别指望它能把你的整个项目历史都背下来。

有人管这叫"context engineering"，听着挺高大上。其实就是学会怎么跟这只智能怪兽说话，让它能记住该记住的，忘掉该忘掉的。

### 五

总之，Claude Code这东西，用好了是个宝贝，用不好就是个麻烦。它不像那些会主动给你建议的AI，它更像是个沉默的工人，你让它干什么它就干什么，哪怕你要它去撞墙，它也会认真地撞给你看。

所以用它的第一要诀就是：想清楚你要什么。别指望它会提醒你"这样不对"，它只会按照你说的去做，至于后果，那就是你自己的事了。

我二十七岁那年，以为掌握了工具就能掌握世界。后来才明白，工具只是工具，重要的是用它的人。Claude Code是一只强大的智能怪兽，但驾驭它的是你的手，你的心，你的脑子。

看完本文不够， 你还要入手购买， 入心 入脑 这才算够了。
