# 玩转通用智能体之Claude Code

## 第三章 基本用法：与智能怪兽共舞

装是装上了，真正的考验这才开头。用 Claude Code，像学一支新舞——你得拿捏好分寸：何时进，何时退，何时让它领着走，何时自己带节奏。

### 一

最基本的用法，就是直接跟它说话。

想让它干什么，张口用大白话讲就是。不用记什么稀奇的命令，不用背什么语法，跟跟人搭话没两样。

比方说：

```
> 帮我找出这个项目的入口文件
> 这个函数是做什么的？
> 给我解释一下这段代码
```

它就去翻你的代码，再把答案递回来。听着没什么稀罕，可你想想从前：要找一个项目的入口，得自己一层层翻目录、看 `package.json`、揪那个 `main` 字段。如今一句话的事，就有人——好吧，有只怪兽——替你办了。

### 二

但 Claude Code 真正的能耐，不在答疑，而在动手。

你尽可以吩咐它：

```
> 给这个函数加上错误处理
> 重构这个模块，让它更易读
> 写几个测试用例
```

它便真去改你的代码。不是给建议，不是劝你「该这么做」，而是撸起袖子直接改。改完还回头告诉你：动了哪儿，为何这么动。

这让我想起乡下的木匠。你跟他说「打张桌子」，他不跟你掰扯什么设计理念，不画草图征求意见，抄起锯子就锯。等你回过神，桌子已经支棱起来了。

Claude Code 也是这副做派。

### 三

这做派嘛，有利也有弊。

利在快。你不必手把手教它每一步，只消把目标撂给它，它自会去想辙。像是手底下有个老练的帮手，交代一声就行，用不着你盯着事事过问。

弊在它有时擅作主张。你只当它动一个小地方，回头一看，整个文件都让它翻新了。你觉着这种事该先言语一声，它却觉着「这点小事，不劳你费心」。

所以用它的头一条要诀，是学会圈地。明明白白告诉它「只改这个函数」，或者「别动其他文件」。你要是不发话，它没准儿把整个项目都给你掀个底朝天。

### 四

Claude Code 还有一桩本事：它会动用工具。

它不单是个语言模型，还能调各样工具来替你干活。它能：

- 读文件（Read）
- 搜代码（Grep）
- 跑命令（Bash）
- 改文件（Edit）
- 列目录（Glob）

这就是说，它不只「晓得」你的代码，是真能「上手」你的代码。它能跑测试，看改动出没出岔；能把整个项目搜一遍，把相干的地方一处不落地揪出来；能一口气改一摞文件，把重构一次办妥。

这就好比，你不光雇了个机灵的程序员，还顺带配给他一整套趁手的家伙。

### 五

可这一身本事，都有个前提：你得会跟它说话。

Claude Code 厉害归厉害，到底不会读心。你说得越清楚，它做得越漂亮；你说得含糊，它就只好瞎猜，猜岔了你又要恼。

所以，会写好提示词，是用好 Claude Code 的关窍。这事要紧，下一章专门来讲。
