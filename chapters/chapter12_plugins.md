# 玩转通用智能体之Claude Code

## 第十二章 Plugins：插件的力量

我年轻时使过一个编辑器，叫 Vim。这玩意儿本身朴素得很，可有一套插件的路子：你能装上各色插件，把它捯饬成一个趁手的 IDE。我记得装过语法高亮的、代码补全的、文件浏览的，林林总总几十个，把我那台 Vim 拾掇得独一份儿。

Claude Code 也有 Plugins（插件），路数跟 Vim 不尽相同，可心思是一样的：扩本事，合脾胃。

### 一

Plugins 和 Skills，差在哪儿？

Skills 多半是讲门道、立规矩，告诉怪兽在某种场合该怎么做。Plugins 更偏向添家伙——给它配上新能耐，或者把一整套配置打个包，一并装上。

打个比方：Skills 像给伙计做培训，Plugins 像给伙计配一套新工具。

实则一个插件里头，往往把技能、子代理、Hooks、MCP 服务一股脑都捎上——装一个插件，等于把人家配齐的一整套家什连锅端了过来。

### 二

插件怎么装、怎么用？

总入口是 `/plugin` 这个命令。插件大多搁在一个个「集市」（marketplace）里——你先把集市添上，再从里头挑插件装。装妥了，`/plugin` 里就能开开关关、卸了重装。

集市这东西，可以是官方的，也可以是社区的，甚至是你自家攒的一个 Git 仓库。你要什么能耐，就去寻对应的插件来装。

### 三

Plugins 的好处是能伸能展。

官方不可能把天下功能都塞进 Claude Code，插件这条路，让社区能你一件我一件地往上添。乐高积木就是这么个理儿：基础的块是有数的，可拼起来能搭出千百样东西。

### 四

可 Plugins 也有它的麻烦。

头一桩是良莠不齐。官方的插件大抵靠得住，第三方的就难说了——有的带 bug，有的没人照看，有的甚至不大干净。

第二桩是搭不搭。插件彼此可能打架，插件跟 Claude Code 的版本也可能对不上。

所以装插件得拿捏：

- 先紧着官方的用
- 装第三方的之前，先看看口碑和更新勤不勤
- 别装太多，够使就成

这就像吃药：能治病，可也有副作用，不能由着性子乱吃。

### 五

说到底，Plugins 是把 Claude Code 越用越顺手的一条路。挑得在理，装得有度，它就帮你长本事；贪多嚼不烂，反倒添乱。
